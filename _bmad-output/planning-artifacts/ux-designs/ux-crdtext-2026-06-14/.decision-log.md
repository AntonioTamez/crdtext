## DL-UX-007 — Dirección visual seleccionada: Editorial Tech (B)
**Decisión:** Dirección B — Editorial Tech. Serif (Georgia) en el cuerpo del editor, monospace en datos técnicos, sans-serif para el chrome UI. Azul (#2563eb light / #3b82f6 dark) como acento principal. Light y dark como ciudadanos de primera clase.
**Razón:** El usuario eligió explícitamente la dirección B.
**Fecha:** 2026-06-14

## DL-UX-008 — Asignación de colores de usuarios
**Decisión:** Usuario A: azul (#2563eb / #60a5fa); Usuario B: violeta (#7c3aed / #a78bfa). ✅ Confirmado por el usuario.
**Fecha:** 2026-06-14

## DL-UX-009 — Sin sistema de componentes externo
**Decisión:** No se usa shadcn/ui ni ningún design system externo — todos los tokens se definen desde cero. ✅ Confirmado por el usuario.
**Fecha:** 2026-06-14

## DL-UX-010 — Layout: editor izquierda, visualizador sidebar derecha
**Decisión:** Panel de operaciones como sidebar colapsable a la derecha. Topbar permanente con logo, identidad de usuarios y badge de conectividad. ✅ Confirmado por el usuario.
**Fecha:** 2026-06-14

## DL-UX-012 — Sin keyboard shortcut para viz toggle en MVP
**Decisión:** No hay atajo de teclado para abrir/cerrar el panel del visualizador en MVP. Solo click. ✅ Confirmado por el usuario.
**Fecha:** 2026-06-14

## DL-UX-013 — Cursor overlays via contenteditable + Range API
**Decisión:** El editor usa `contenteditable` (no `<textarea>`) para permitir cursor overlays posicionados con Range API. ✅ Confirmado por el usuario.
**Fecha:** 2026-06-14

## DL-UX-014 — Colores de tags de operación: INSERT=azul, DELETE=rojo
**Decisión:** INSERT tag usa azul (`{colors.user-a}`), DELETE usa rojo (`{colors.status-offline}`), independientemente del site que generó la operación. El site se identifica por el texto "site:ID" en la misma línea. ✅ Confirmado por el usuario.
**Fecha:** 2026-06-14

## DL-UX-011 — Animación del panel: smooth slide
**Decisión:** El panel del visualizador se abre y cierra con una animación suave (smooth slide). No es instantáneo.
**Razón:** Confirmado explícitamente por el usuario — reemplaza el [ASSUMPTION] de apertura instantánea.
**Cómo aplicar:** width transition de 300ms ease-in-out sobre la columna derecha del grid. El contenido del panel hace fade-in con 50ms de delay para no aparecer antes de que el panel esté abierto.
**Fecha:** 2026-06-14

## DL-UX-006 — Todas las pantallas son importantes
**Decisión:** No hay una pantalla más importante que otra — el editor, el visualizador de operaciones, los cursores y el badge de conectividad son todos piezas clave del demo.
**Fecha:** 2026-06-14
